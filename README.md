## üèõÔ∏è The Four Pillars of Quasar

### 1. Mode::Turbo (The Speed King)
*   **Speed**: > 5.2 GB/s
*   **Philosophy**: Raw speed. No validation. No comments.
*   **Architecture**: Pure View-based parsing using a structural bitmask generated by AVX2/AVX-512 intrinsics. It slices memory without moving it.
*   **Optimization**: Completely branchless hot-path.

### 2. Mode::Apex (The Glaze-Killer)
*   **Speed**: High-Throughput Typed Parsing
*   **Philosophy**: Zero-Materialization.
*   **Technology**: "Direct-Key-Jump" algorithm. Uses `*(uint64_t*)` casts to compare 8 bytes of a key instantly against the input stream, skipping entire sub-trees using the structural bitmask.

### 3. Mode::Standard (The Balanced Warrior)
*   **Speed**: ~3.5 GB/s
*   **Philosophy**: Faster than Simdjson's DOM, but easy to use.
*   **Features**: Full JSONC support, basic structural validation.

### 4. Mode::Titan (The Tank)
*   **Speed**: ~3.5 GB/s (with Validation)
*   **Philosophy**: Maximum Safety.
*   **Hardening**:
    *   **AVX-512 UTF-8 Validation**: Uses `vpcmpeqb` and `vpand` logic.
    *   **Strict Bounds**: Implements masked loads (`_mm512_maskz_loadu_epi8`) at buffer edges to prevent segfaults even on truncated input.

## üß† Architecture: The Dual-Engine

Tachyon detects your CPU at runtime and hot-swaps the parsing kernel.

### AVX2 Engine
*   **Throughput**: 32 bytes/cycle.
*   **Technique**: Uses `vpshufb` (shuffle) tables to classify characters into structural types (Brace, Bracket, Quote, Colon, Comma) in a single step.
*   **Memory**: Software prefetching (`_mm_prefetch`) hides memory latency on huge files.

### AVX-512 Engine ("God Mode")
*   **Throughput**: 64 bytes/cycle.
*   **Technique**: Leverages `opmask` registers (`k0`-`k7`).
*   **Instruction Set**:
    *   `_mm512_cmpeq_epi8_mask`: Direct comparison to mask.
    *   `_mm512_maskz_loadu_epi8`: Safe tail loading.
*   **Transition Safety**: Explicit `_mm256_zeroupper()` calls prevent AVX-SSE transition frequency penalties.
